++ #[[# Num]]

[[div style="margin-left: 35px;"]]

: **68K [[# 68k]]** : Motorola 68000 is the microprocessor that the TI-89, TI-92, and Voyage 200 graphing calculators use. Zilog makes the microprocessor for the TI-83 series of  graphing calculators.

[[/div]]

++ A[[# A]]

[[div style="margin-left: 35px;"]]

: **[[[Advanced Basic]]] [[# advancedbasic]]** : Quality TI-Basic programming that is not only efficient in terms of size and speed, but is used to make genuinely fun games.

: **AI [[# ai]]** : An abbreviation for artificial intelligence. Commonly used when referring to an opponent in a game that is not another player.

: **[[[Ans]]] [[# ans]]** : The last answer that was stored in the calculator. The calculator also keeps a history of the last few statements entered in on the home screen where general  usage of the calculator occurs.

: **Application (//App//) [[# application]]** : A specialized assembly program that is stored in [#flash Flash] ROM and is run through the APPS menu. There are ten 16KB pages available for Flash Applications on the TI-83+, while there are ninety-four 16KB pages available for Flash Applications on the TI-83+SE.

: **Archive (//Arc//) [[# archive]]** : Moving a variable or program from RAM to Flash ROM (or just ROM). Archive is the permanent memory (also known as ROM) on the TI calculator. If your calculator crashes, all of your files in RAM will be lost, while everything in the archive will still be intact.

: **Argument (//Arg//) [[# argument]]** : One of the values that a [#command command] acts on, or an expression that is supplied as input to a program, function, or command.

: **Array [[# array]]** : A collection of numeric [#variable variables] arranged in a [#list list] or [#matrix matrix] for processing by the calculator. Each element in an array is referenced by its [#index index] - position in the array.

: **ASCII [[# ascii]]** : The American Standard Code of Information Interchange. It uses 1 [#byte byte] (8 [#bit bits]) to hold a [#character character], and thus is limited to 256 different characters. The calculator uses a modified version of ASCII internally to store characters, but the TI-Basic interpreter deals with tokens instead.

: **[[[Assembly]]] (//Asm//) [[# assembly]]** : The other programming language built-in to the TI graphing calculators, apart from [#tibasic TI-Basic]. It is much more difficult to learn and program in, but at the same time it is much more powerful, and can be used to create very sophisticated games and [#program programs].

[[/div]]

++ B[[# B]]

[[div style="margin-left: 35px;"]]

: **Backup [[# backup]]** : Copying files from the calculator to a computer, or creating a second copy of the files on the calculator. It is usually done to prevent the person from losing the files through an inadvertent RAM clear.

: **Benchmark [[# benchmark]]** : A reference point or standard against which speed and/or size can be compared. It is useful when optimizing a program.

: **[[[binandhex|Binary]]] [[# binary]]** : The two-digit ([#bit bit]) [#number-system number system] based on 0 and 1, similar to the way the normal [#decimal decimal] system is based on the numbers 0-9. For example the number 101 in binary represents 1*2^^2^^+0*2+1, or 5 in decimal.

: **Bit [[# bit]]** : A [#binary binary] digit (0 or 1).

: **Boolean [[# boolean]]** : A logic system in which conditions evaluate to either true or false.

: **Branch [[# branch]]** : A departure from the sequential performance of program statements. An unconditional branch causes the calculator to jump to a specified place in the program each time the branching statement is encountered. A conditional branch transfers program control based on the result of some logical [#condition condition].

: **Breakpoint [[# breakpoint]]** : A point in a program where program execution can be suspended, used when [#debugging debugging] the program. In [#tibasic TI-Basic], such breakpoints can be created with the [[[Pause]]] command.

: **Bug [[# bug]]** : An error in a [#program program] (see also [#debugging Debugging]).

: **Byte [[# byte]]** : A string of eight binary bits. The calculator uses one byte of information to encode the letter "A", as well as most [#token tokens]. Bytes (or kilobytes - one kilobyte is 1024 bytes) are used to measure the amount of memory a program or variable takes up on the calculator.

[[/div]]

++ C[[# C]]

[[div style="margin-left: 35px;"]]

: **Computer Algebra System (//CAS//) [[# cas]]** : A feature available on some of the [[[nspire:home|Ti-Nspire]]] calculators. It can symbolically solve equations, factor and expand variable expressions, complete the square, find antiderivatives, compute limits and exact solutions in irrational forms, and more.

: **Calculator Based Laboratory (//CBL//) [[# cbl]]** : A device that you connect to the calculator through the link port, and it collects real-time data from various sensors that you can connect to it, and allows you to view and analyze the results.

: **Calculator Based Ranger (//CBR//) [[# cbr]]** : A device that you connect to the calculator through the link port, and it collects real-time data from various sensors that you can connect to it, and allows you to view and analyze the results.

: **Cascading Style Sheet (//CSS//) [[# css]]** : A type of code that changes how a website looks based upon an element's class, id, name, or type.

: **Certificate [[# certificate]]** : A certificate is a file that is generated by TI, and when loaded on the calculator provides needed information to Flash applications on the calculator.

: **Character (//Char//) [[# character]]** : A letter, number, punctuation symbol, or special graphics symbol. They are encoded in [#ascii ASCII] format internally, but the [#tibasic TI-Basic] interpreter uses [#token tokens] instead.

: **Clock cycle [[# clock-cycle]]** : The unit of time in a [#cpu CPU]. It is so small that even the most primitive [#assembly assembly] instructions take around 10 clock cycles to run; the amount of clock cycles it takes for a single [#tibasic TI-Basic] command can be in the millions.

: **Command (//Cmd//) [[# command]]** : A word or token that tells the calculator to do something. Examples: [[[ClrHome]]], [[[Output(]]], [[[While]]], etc. They are sometimes subdivided into [#function functions] and [#instruction instructions]

: **Complex [[# complex]]** : Any expression or number which has an imaginary component (see //[[[i]]]//). For example, 3 + 2i is a complex number, where //i//=sqrt(−1).

: **Concatenation [[# concatenation]]** : Joining two or more [#string strings] together to make a longer string. The "+" is the concatenation operator. For example, "TI-"+"BASIC"="TI-BASIC".

: **Condition (//Cond//) [[# condition]]** : A logical expression that the calculator can evaluate to be true or false (represented as 1 and 0).

: **Constant [[# constant]]** : A specific number, [#list list], [#matrix matrix], or [#string string] that is given as a fixed value in the program (such as 9.024 or "Hello"), rather than contained in a [#variable variable].

: **Central Processing Unit (//CPU//) [[# cpu]]** : The "brain" of the calculator, where the calculator controls the processing and execution of everything.

: **Crash [[# crash]]** : A sudden failure of the calculator that is usually caused by a faulty assembly program. Although TI-Basic programs can crash, the majority of TI-Basic errors are caught by the calculator.

: **Cursor [[# cursor]]** : A small, flashing square showing where a typed character will appear. A cursor (in the form of a crosshair) is also used on the graph screen to input a point from the user.

[[/div]]

++ D[[# D]]

[[div style="margin-left: 35px;"]]

: **Data [[# data]]** : Information, often numerical in form. When data is used by a calculator program, it can be [#hardcode hardcoded] or [#softcode softcoded].

: **Debugging (//Debug//) [[# debugging]]** : Fixing a [#program program] to remove [#bug bugs], or mistakes. There are many [[[debug|techniques]]] for debugging, some universal, some specific to calculator programming.

: **Decimal (//Dec//) [[# decimal]]** : A base 10 [#number-system number system] using the symbols 0-9. It is the only number system natively supported by the calculator.

: **Default [[# default]]** : A standard characteristic or value which the calculator assumes without specification. For example, the [[[round(]]] command has a default of rounding 9 digits past the first digit, so round(X) will round X to 9 digits. This can be overwritten with a second [#argument argument] to specify the digits to round to -- round(X,3), for example, rounds to 3 digits.

: **Display [[# display]]** : The home screen or graph screen on the calculator.

[[/div]]

++ E[[# E]]

[[div style="margin-left: 35px;"]]

: **Emulator [[# emulator]]** : A computer program that simulates a calculator's hardware, allowing you to run TI-Basic and assembly programs on your computer.

: **Equation Operating System (//EOS//) [[# eos]]** : The Equation Operating System (EOS^^TM^^) is the routine in the calculator that determines order of operations.

: **Equation (//Equ//) [[# equation]]** : An equation is a mathematical expression involving variables and/or functions.

: **Error (//Err//) [[# error]]** : An error is when an unexpected condition occurs, and the calculator displays a message to attempt to tell you what went wrong. Errors run the gamut from serious to stupid, meaning it might be time to buy a new calculator or you might have simply misused a command.

: **Execute [[# execute]]** : Another name for running a program or command. 

: **Exponent [[# exponent]]** : A number indicating the power to which a number or expression is to be raised, usually written at the right and above the number. For example, 2^^6^^ = 2x2x2x2x2x2. In scientific notation, the power of ten which the mantissa is multiplied by to calculate the actual number.

: **Expression (//Expr//) [[# expression]]** : A combination of numbers, variables, operators, and functions. For example, 5+3, sin(cos(X)), and X are all expressions.

: **eZ80 [[# ez80]]**: The Zilog eZ80 is the chip used in the TI-84+CE. If an [# assembly] program is said to be in eZ80, then it will only work on the CE, an eZ80 calculator.
[[/div]]

++ F[[# F]]

[[div style="margin-left: 35px;"]]

: **Flash [[# flash]]** : The ROM/archive memory on the TI-83+/84+/SE calculators where applications are stored. It is also used to describe the technology employed in the calculator enabling increased storage, upgradeability, and expandability.

: **Floating Point [[# floatingpoint]]** : A method used to represent numbers with non-zero fractional components, where a number is represented as a signed mantissa implicitly multiplied by a signed exponent.

: **Flow Chart [[# flowchart]]** : A diagram of geometric shapes connected by arrows that show the progression of a program. Flow charts are handy for developing complicated programs and illustrating how programs work.

: **Fraction (//Frac//) [[# fraction]]** : A fraction is the the result of division of two numbers (i.e., 5/7 or 1/3), including integers or even other fractions.

: **[[[Friendly Window]]] [[# friendlywindow]]** : A configuration of window variables that's most useful for your program -- most commonly, because it makes the coordinate value of a pixel come out to a round value, such as an integer, or .1 of an integer.

: **Function (//Func//) [[# function]]** : Although the word has many meanings, in the context of TI-83 series calculator programming, it's a type of [#command command] that returns a value -- such as [[[gcd(]]] or [[[sin(]]].

: **FX [[# fx]]** : Graphical or sound effects used in a game to make it stand out.

[[/div]]

++ G[[# G]]

[[div style="margin-left: 35px;"]]

: **Garbage Collection [[# garbagecollection]]** : A process that the calculator automatically performs to reorganize the contents of Flash ROM. When you delete a file from Flash ROM, it is not deleted at that time, it is just marked as 'not used'. When the Flash ROM is full, the variables marked as deleted are actually erased.

: **Graphics [[# graphics]]** : Any sort of visual display on the screen, such as graphs, patterns, and drawings, both stationary and animated.

: **Graph Link [[# graphlink]]** : The link cable that is used together with TI-Connect or the Graph Link software to send programs, variables, Flash applications, and operating systems.

: **Graph Screen [[# graphscreen]]** : The display screen on which graphs and plots are shown.

: **Group [[# group]]** : A collection of files that have been combined into one file, stored in Flash ROM.

: **Graphical User Interface (//GUI//) [[# gui]]** : A type of user interface which allows people to interact with the calculator.

[[/div]]

++ H[[# H]]

[[div style="margin-left: 35px;"]]

: **Hardcode [[# hardcode]]** : Hardcoded [#data data] is information used by the program that results from the logic of its [#command commands]. It is very easy for the program to read, but can be impossible to change.

: **Hardware [[# hardware]]** : The circuit boards and other electronic parts which make up a calculator.

: **[[[binandhex|Hexadecimal]]] [[# hexadecimal]]** : A base 16 [#number-system number system] using 16 symbols, 0-9 and A-F. It is used as a convenient shorthand way to express [#binary binary] code, because every four bits of binary have a corresponding hexadecimal symbol.

: **Home Screen [[# homescreen]]** : The default display screen on which calculations and commands are entered.

[[/div]]

++ I[[# I]]

[[div style="margin-left: 35px;"]]

: **Integrated Development Environment (//IDE//) [[# ide]]** : An IDE is a software application that provides utilities for program development, usually including an editor and debugger.

: **Image (//Img//) [[# image]]** : An image is a visual depiction of  something.

: **Input/Output (//I/O//) [[# io]]** : Input is what you provide to the calculator when you press a key or select a menu, while the output is how the calculator responds. The two are interdependent of  each other.

: **Index [[# index]]** : A position in an [#array array]. In TI-Basic, arrays are indexed starting with 1 (so that the first element in a [#list list] is numbered 1, the second 2, and so on). In some other languages, they are indexed starting with 0.

: **Input [[# input]]** : The means by which data is entered into the calculator by the [#user user], primarily through the calculator's keys (though the link port is also a form of input).
 
: **Instruction [[# instruction]]** : In the case of TI-83 series programming, this is often used to indicate a type of [#command command] that does not return a value, but stands alone: for example, [[[Line(]]], or [[[Input]]]. It is also sometimes used to describe commands in general.

: **Integer (//Int//) [[# integer]]** : A whole number, either positive, negative, or zero.

: **Interpreter [[# interpreter]]** : The algorithm stored inside the calculator that runs a [#tibasic TI-Basic] program by executing the appropriate system routines.

: **Iteration [[# iteration]]** : One repetition of a [#loop loop]. For certain [#command commands], the calculator has an internal loop for which the total number of iterations is relevant, and affects accuracy.

[[/div]]

++ K[[# K]]

[[div style="margin-left: 35px;"]]

: **Keypad [[# keypad]]** : The panel of keys used to enter [#program programs] and [#data data] into the calculator.

[[/div]]

++ L[[# L]]

[[div style="margin-left: 35px;"]]

: **Library (//Lib//) [[# lib]]** : A library is a program (usually made in Assembly) that is designed to be an auxiliary utility for another program. Libraries provide functions that are not supported or not practical in TI-Basic (such as inverting the screen).

: **[[[lists|List]]] [[# list]]** : A sequence of numerical [#variable variables] which can be accessed and modified by their position.

: **Loop [[# loop]]** : A group of one or more consecutive program lines which are performed repeatedly, either a specific number of times or until a [#condition condition] is met.

: **Lua [[# lua]]** : A programming language other than Ti-Basic which can be used on the [[[nspire:home|Ti-Nspire]]] calculators

[[/div]]

++ M[[# M]]

[[div style="margin-left: 35px;"]]

: **Mantissa [[# mantissa]]** : The basic numeric portion of a number expressed in scientific notation. In 3.264E + 4, the mantissa is 3.264.

: **[[[matrices|Matrix]]] (//Matr//) [[# matrix]]** : A two-dimensional grid of numerical [#variable variables] which can be accessed and modified by their position.

: **Memory [[# memory]]** : The two different places (RAM and ROM) where calculator [#program programs] and [#data data] are stored. All variables and programs take up part of memory to store, which makes the size of a program or variable very important.

: **[[[memory-leaks|Memory Leak]]] [[# memoryleak]]** : A type of [#bug bug] in which the program wastefully uses memory, often culminating with an error. In TI-Basic, this type of bug is often due to exiting a [[[while|loop]]] or [[[If]]]-Then-End block with a [[[Goto]]].

: **MHz [[# mhz]]** : The speed the calculator [#cpu CPU] runs at. For the TI-83, it is 6MHz, while for the TI-83+ it is 8MHz, and for the TI-83+SE and TI-84+/SE calculators, it is 15MHz. It's measured in millions of [#clock-cycle clock cycles] per second.

[[/div]]

++ N[[# N]]

[[div style="margin-left: 35px;"]]

: **Number system [[# number-system]]** : A way of writing down a number. Two common systems used by people are the [#decimal decimal] (Arabic) system, and the Roman numeral system. The calculator uses the [#binary binary] system internally, but the usual decimal system is all that it shows or accepts as [#input input].

[[/div]]

++ O[[# O]]

[[div style="margin-left: 35px;"]]

: **Operator (//Op//) [[# operator]]** : A symbol used in calculations (numeric operators) or in relationship comparisons (related operations). The math operators are +, -, *, /, ^. The relational operators are >, <, =, ≥, ≤, ≠. The logic operators are and, or, xor, not(.

: **Optimization [[# optimization]]** : The process of improving the running time, length, or memory usage of a program.

: **Order of Operations [[# orderofoperations]]** : When evaluating an expression, the operators are calculated in a specific order defined by the OS.

: **Output [[# output]]** : Information sent from the calculator, e.g. graphics on the screen. Also, the means by which data leaves the calculator, through either the link cable, graph link, or USB cable.

: **Operating System (//OS//) [[# os]]** : The internal program that runs the calculator and includes all the functionality needed to use the calculator.

: **On-Calculator (//on-calc//) [[# oncalc]]**: This refers to something that is 100% on your calculator; not requiring external tools.

[[/div]]

++ P[[# P]]

[[div style="margin-left: 35px;"]]

: **Port [[# port]]** : Convert; Remake, especially onto another device.

: **Program (//Prgm//) [[# program]]** : The list of instructions that tells the calculator what to do to perform a task.

: **Programmer [[# programmer]]** : A person who writes programs.

: **Programming Language [[# programminglanguage]]** : Numeric or alphabetic commands which the calculator can understand, and execute.

: **[[[Prompt]]] [[# prompt]]** : A command that requests input from the user.

: **Pt [[# pt]]** : On the monochrome calculators, a point is a specific 1x1 space on the calculator's graph screen. You can use the [[[pt-on|Pt commands]]] to manipulate a point. The primary difference between a pixel and a point is that a pixel is not affected by the graph screen settings, while a point is.

: **Pxl [[# pxl]]** : A pixel is a specific 1x1 space on the calculator's graph screen. You can use the [[[pxl-on|Pxl commands]]] to manipulate a pixel.

[[/div]]

++ R[[# R]]

[[div style="margin-left: 35px;"]]

: **Random Access Memory (//RAM//) [[# ram]]** : A temporary memory, i.e. one in which data is stored so long as electrical power is applied. Data in RAM can be accessed or changed and is lost if the batteries are removed from the calculator. 

: **[[[Rand]]] [[# rand]]** : A command used to generate a  uniformly-distributed pseudo-random number between 0 and 1.

: **[[[Recursion]]] [[# recursion]]** : A program that calls itself as a subroutine (or a function that's defined in terms of itself). If not done carefully, this can result in a [# memory-leak] See [#recursion recursion].

: **Read Only Memory (//ROM//) ( [[# rom]]** : Certain instructions for the calculator are permanently stored in ROM and can be accessed but cannot be changed. [#data Data] in ROM is not lost if batteries are removed.

: **[[[routines|Routine]]] [[# routine]]** : A small section of code intended to perform a specific function. Typically the most optimized form of the code, and used quite often.

[[/div]]

++ S[[# S]]

[[div style="margin-left: 35px;"]]

: **Scientific Notation [[# scientificnotation]]** : A method of expressing very large or very small numbers by using a base number ([#mantissa mantissa]) times ten raised to some power ([#exponent exponent]).

: **Scroll [[# scroll]]** : Shifting part of the screen to give the illusion of seeing only part of an image larger than the screen.

: **SE [[# se]]** : Silver Edition is the more powerful version of the TI-83+ and TI-84+ calculators, with more memory and a faster processor.

: **Shareware Application [[# shareware]]** : An application that is signed shareware is one that can be loaded and executed on all calculators for which the application was developed.

: **Shell [[# shell]]** : A program, typically programmed in assembly, that is used to run other programs. The majority of assembly programs require that they be run from a shell.

: **Signing [[# signing]]** : Signing is a process whereby a file (*.hex on the TI-83+ and *.cer on the TI-89/TI-92+/V200) is made into either a shareware or a secured application by TI.

: **Softcode [[# softcode]]** : Softcoded [#data data] is information stored in an [#array array] at the beginning of the program, and read from it later. It is usually slower to read than [#hardcode hardcoded] data, but the advantage is that the same code can often be used for different data stored in the same way.

: **Software Development Kit (//SDK//) [[# sdk]]** : A Software Development Kit (SDK) enables software developers to write applications (apps) using a variety of the calculator's internal functions: simulator/debugger, freeware private key (gives the ability to digitally sign a Flash Application for use on any TI-83+/SE calculator), and digital signature utilities (all the utilities needed to sign a Flash Application).

: **[[[Spaghetti Code]]] [[# spaghetticode]]** : The unofficial name given to code that heavily relies on [[[Goto]]]/[[[Lbl]]] for its structure and organization.

: **Sprite [[# sprite]]** : A small image which is moved around the screen using code and/or repeated multiple times on the screen.

: **Stack [[# stack]]** : A [#data data structure] much like a list but with limited access: only the top element at a given time can be accessed or changed. The [#os OS] uses two stacks internally for running TI-Basic programs: one for evaluating expressions, and one for keeping track of Ends and program calls.

: **Statement [[# statement]]** : A single line of a program containing a single instruction such as [[[ClrHome]]], [[[Output(]]], [[[Stop]]], etc. Usually synonymous with [#command command].

: **[[[strings|String]]] (//Str//) [[# string]]** : A variable type that stores text as a series of symbols or [#token tokens].

: **[[[Subprogram]]] [[# subprogram]]** : A program segment which can be used more than once during the execution of a program, such as a complex set of calculations or a print routine. You can make subprograms using the [[[prgm]]] command.

: **Syntax [[# syntax]]** : The rules that the TI calculator follows when typing in a command. You need to type in the command exactly as specified, otherwise, you will get a [[[errors#syntax|ERR:SYNTAX]]] error.

[[/div]]

++ T[[# T]]

[[div style="margin-left: 35px;"]]

: **Texas Instruments (//TI//) [[# ti]]** : The makers of the TI graphing calculators. In addition to graphing calculators, they also make a large assortment of other electronic devices.

: **TI-83/84/+/SE [[# ti83]]** : Shorthand way to refer to the five graphing calculators in the TI-83 series: TI-83, TI-83+,  TI-83+SE, TI-84+, and TI-84+SE.

: **TI-Basic [[# tibasic]]** : The official name of the programming language found on TI calculators.

: **TI-OS [[# tios]]** : The built-in operating system that controls the operation of the calculator.

: **Token [[# token]]** : A single instruction or symbol in TI-Basic. Anything you type into a program is made up of tokens. The more common tokens take up one byte each, but others can take up two. Some tokens are not accessible through menus, and require assembly to gain access to, but can then be used in Basic programs.

[[/div]]

: **User [[# user]]** : When talking about the effect of a program or command, it is the person running the program or command on his/her calculator.

There are several different variables available in TI-Basic, including reals,  lists, matrices, and strings.
